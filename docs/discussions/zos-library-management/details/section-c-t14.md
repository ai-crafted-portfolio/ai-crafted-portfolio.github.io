# §C-T14. 通知 / ドキュメント強制 (4 機能)

Change Tracker (S15) の通知 / ドキュメント強制 4 機能。

---

## 論点

メール送信機能を採用するか、社内連絡ルールに任せるか。

## 個別判定

| # | 機能 | 判定 | 実装方法 / 割切理由 |
|---|---|---|---|
| 1 | CSAVE (変更理由記録) | 採用 | Excel 列「変更理由」 |
| 2 | LOGCOMMENT (運用ログ) | 採用 | Excel 履歴シートにコメント列 |
| 3 | EMAIL (メール通知) | 割切 | PC 側メールクライアント連携が要、社内連絡ルールに委ねる |
| 4 | ISPF EDIT 警告 | 割切 | ISPF Edit エグジット利用不可 (z/OS 内部処理) |

!!! success "T14 の判定結果"
    **4 機能のうち 2 機能採用。** メール通知と ISPF Edit 警告は外部依存・技術制約で割切。
