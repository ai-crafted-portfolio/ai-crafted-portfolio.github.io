# §A. 6 領域の判定詳細

[全体観 §2 「6 領域分類」](../overview/02-domains.md) の各領域について、対象/対象外の判定理由を述べる。

判定の枠組みは [SSOT ③](../overview/01-ssot.md) (公式の変更管理体系で完全にはカバーされていない領域を対象とする基準) を適用したもの。

---

## A-1. IBM 保守領域 + システム制御 + システムプロシージャの判定

### IBM 保守領域

| 項目 | 内容 |
|---|---|
| 対象範囲 | z/OS の起動時に必須のシステムライブラリ群 (SYS1.NUCLEUS 等) |
| 重要度 | 高 |
| 判定 | 本ツール対象外。SMP/E に管理を委ねる |
| 判定根拠 | 公式マニュアルで「IPL 時に決定され never be changed or removed」と明記 |
| 出典 | S2 (zOS31_ieae200) |
| 根拠強度 | 強 |

### システム制御領域

| 項目 | 内容 |
|---|---|
| 対象範囲 | z/OS の動作を制御する設定パラメータ群 (PARMLIB の IEASYSxx, SMFPRMxx, COMMNDxx 等) |
| 重要度 | 中 |
| 判定 | **本ツール対象 (PARMLIB として)**。運用所管メンバ (COMMNDxx 等) と運用に直接影響するメンバ (LOADxx, IEASYSxx 等) を含む |
| 判定根拠 | 公式の変更管理体系では完全にカバーされず、サイト運用ルールに依存している部分が多い (Common Criteria S13 で SMFPRMxx が監査対象と明記) |
| 出典 | S2, S13 (Common Criteria) |
| 根拠強度 | 中 |

### システムプロシージャ領域

| 項目 | 内容 |
|---|---|
| 対象範囲 | z/OS のシステム標準プロシージャ群 (SYS1.PROCLIB) およびサイトカスタム PROC |
| 重要度 | 中 |
| 判定 | **本ツール対象 (PROCLIB として)**。サイトカスタム PROC + JES2/VTAM 等の起動 PROC を含む |
| 判定根拠 | JES2 起動時に決定される構造 (S3, S4)。公式の変更管理体系では完全にカバーされず、サイト運用ルールに依存 |
| 出典 | S3 (zOS31_ieab600), S4 |
| 根拠強度 | 強 |

## A-2. アプリ部門領域 + 運用資材領域

### アプリ実行領域

| 項目 | 内容 |
|---|---|
| 対象範囲 | アプリのロードモジュール (USER.LOADLIB) |
| 重要度 | 中 |
| 判定 | 本ツール対象外。バイナリ差分はアプリソース領域で追跡することで補える |
| 判定根拠 | RECFM=U で人間可読でなく、Excel で差分を見ても理解不能 |
| 出典 | S6, S11 |
| 根拠強度 | 強 |

### アプリソース領域

| 項目 | 内容 |
|---|---|
| 対象範囲 | アプリ部門のソースコードと JCL |
| 重要度 | 中 |
| 判定 | 本ツール対象外。既存 SCM に管理を委ねる |
| 判定根拠 | SCLM Project Manager Guide で SCM 中核領域として定義 |
| 出典 | S8 |
| 根拠強度 | 強 |

### 運用資材領域 ★本ツールの対象★

| 項目 | 内容 |
|---|---|
| 対象範囲 | 運用部門が直接編集する JCL / REXX / PROC 等、および運用パラメータ・ISPF パネル・SORT/IDCAMS カード等 |
| 重要度 | 高 |
| 判定 | **本ツール対象**。Change Tracker の代替として軽量版を構築 |
| 判定根拠 | S5 / S13 / S17 で公式管理空白を確認
