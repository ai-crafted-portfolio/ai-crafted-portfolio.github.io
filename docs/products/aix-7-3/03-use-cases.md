# AIX 7.3 — 典型ユースケース

AIX 7.3 — 典型ユースケース（運用手順）

| ユースケース名 | 想定状況 | 手順サマリ | 使用コマンド | 注意点 | 出典 |
|---|---|---|---|---|---|
| 新規 OS インストール（上書き） | DVD/USB から AIX 7.3.3 を初期導入 | 1) ハードウェア要件確認（POWER8+, 2GB+, 20GB+）<br>2) インストール媒体作成（DVD コピーまたは `dd if=/dev/cd0 of=/dev/usbms0 bs=4k`）<br>3) SMS メニューでブート対象選択（最初の 144 I/O slot 内に boot device）<br>4) BOS Install メニューで暗号化対象 LV を選択（hd2/hd9var/hd3/hd1/hd10opt/hd11admin/dumplv）<br>5) ライセンス受諾（bos.licenses fileset 同梱）<br>6) SWMA acceptance ウィンドウで accept/decline を回答<br>7) インストール完了後、PKS 認証 passphrase を追加 | dd, bootlist, installp, bosinst.data | hd5 ≥40MB（ディスク先頭 4GB 内、連続パーティション）。NIM の場合 ACCEPT_LICENSES で SWMA 自動承認可 | S35, S18 |
| 既存 OS のマイグレーション（7.1/7.2 → 7.3.3） | 旧版 AIX を 7.3.3 へインプレースマイグレーション | 1) 旧版上で premigration script を実行（/usr/lpp/bos 配下）<br>2) hd5 容量不足の場合、空きパーティション確保<br>3) base media または NIM でマイグレーション開始<br>4) bos.dsc fileset を最初に installp（base media 利用時）<br>5) 旧版 fileset の置換（bos.net.tcp.bind → bind.rte 等） | premigration script, installp -e /tmp/install.log -aXd <src> bos.dsc | Java 8 32bit SR6FP35 はロード不能の場合あり、SR6FP30 へ強制降格。PowerSC Trusted Surveyor (powersc.ts) は事前削除必須（WPAR 含む）。rsct.vsd / rsct.lapi.rte は削除必須 | S35 |
| NIM サーバ導入と LPP_SOURCE 構築 | 複数 AIX クライアントを集中管理 | 1) bos.sysmgt.nim.master fileset 導入<br>2) /usr/lpp/bos.sysmgt/nim/README 参照<br>3) LPP_SOURCE 作成（base media + 必要な update を統合）<br>4) image_data resource 作成（base+update 混在時の必須）<br>5) SPOT 作成、SPOT 用 file system に十分な空き容量を確保<br>6) クライアントを NIM master に登録 | nim, lsnim, lppsrc, mksysb | LPP_SOURCE が base+update 混在の場合、SPOT 作成時に image_data 必須。missing image (network boot image / bos.net.nfs.client / bos.net.tcp.bootp) が出たら SPOT 領域不足を疑う | S35 |
| 暗号化 rootvg 構築 | 起動ディスクを暗号化 | 1) 新規/上書き install 時 BOS install メニューで暗号化対象 LV を選択<br>2) PKS 認証で初期化<br>3) 起動完了後、passphrase recovery method を追加<br>4) hdcryptmgr で運用管理（鍵追加・PKS 移行） | hdcryptmgr, pksctl | TL1 以降 Live Update 対応（nonrootvg のみ）。Power FW1030+ で PKS 移行可。multibos / mksysb (alt_disk_*) で再導入時は PKS+passphrase は再生成 | S35 |
| Live Kernel Update (LKU) 実施 | 業務無停止でカーネル更新 | 1) 互換性確認（TL/SP）<br>2) lvupdate.data 編集（ipsec_auto_migrate=yes 等）<br>3) geninstall (LKU mode) または HMC ベース LKU で実行<br>4) blackout 時間中、アプリは一時停止<br>5) 完了後 audit subsystem (STREAM mode) との互換性確認 | geninstall, oslevel -s | TL3 で性能改善・blackout 短縮・LPAR 動的拡張対応・libc 等の Live Library Update（LLU）導入 | S35 |
| MPIO 既定値の確認・調整 | AIX 7.3 で reserve/algorithm/queue_depth が変更されたため業務影響を確認 | 1) `lsattr -El hdiskN` で現状値確認<br>2) ストレージ仕様（DS8000 / SVC / Flash 別）に合わせ chdev で調整<br>3) -U フラグで動的反映（オープン中も適用可） | lsattr, chdev -U, lspath | clr_q / q_err / q_type / dist_err_pcnt / dist_tw_width / lun_reset_spt / reassign_to / start_timeout は nondisplay 化済（変更には -P 等の管理者権限） | S35 |
| エラーログの確認とアラート設定 | システムエラーの集中監視 | 1) `errpt` で要約、`errpt -a` で詳細表示<br>2) ODM の errnotify クラスでエラー通知設定（label/class/type で条件指定）<br>3) `errclear` で古いエントリ削除<br>4) `snap` でサポート提供データ生成（snap.pax.gz が既定） | errpt, errnotify, errclear, errlogger, errupdate, snap | HW=90日 / SW=30日で自動削除。trace facility は AIX 7.3 で root 限定化 | S45, S35 |
| Cluster Aware AIX リポジトリ構築 | OS レベルクラスタの初期セットアップ（PowerHA 前提条件） | 1) 共有ディスクを確保（NVMe ディスクは TL3 から対応）<br>2) クラスタリポジトリディスクとして指定<br>3) `clmgr add cluster` でクラスタ作成<br>4) RSCT 3.3.0.0 が稼働していることを確認 | clmgr, lscluster, mkcluster | RSCT VSD/LAPI は AIX 7.3 で廃止。compat .sp / .hacmp fileset も shipping 停止（残置されていれば削除可） | S34, S6, S35 |
| WPAR 作成 | OS レベル分離環境を作成し業務アプリを動かす | 1) `mkwpar -n <name>` で system WPAR 作成<br>2) WLM（Workload Manager）でリソース上限設定<br>3) `startwpar` で起動、`clogin` で WPAR 内に入る<br>4) WPAR 内 fileset 構成を確認（PowerSC.ts は削除必須） | mkwpar, startwpar, clogin, lswpar | AIX 7.3 で PowerSC Trusted Surveyor 廃止 → WPAR 内に残存していると migration がブロックされる | S75, S35 |
| OpenSSL 1.0.2/1.1.1 から 3.0 への移行 | 依存アプリケーションの再リンク | 1) 既存 SSL バージョン確認（`openssl version`）<br>2) 旧 /var/ssl/openssl.cnf を退避<br>3) AIX 標準の 3.0 設定ファイル配置<br>4) legacy provider が必要なら openssl_init / legacy_sect を有効化<br>5) アプリケーション再リンクして OpenSSL 3.0 共有オブジェクト（libssl.so.3 / libcrypto.so.3）使用へ | openssl version, openssl ciphers | OpenSSL 1.0.2 EOS 2024/01、1.1 EOS 2025/12、共有オブジェクトは 2025年秋に AIX archive から削除予定 | S35 |
| BIND 9.18 への移行（DNS） | AIX 内蔵 DNS バイナリ更新 | 1) bind.rte fileset インストール<br>2) シンボリックリンク確認（/usr/sbin/named9 → isc_bind/named9）<br>3) named.conf / named-checkconf で構文チェック<br>4) 旧 dnssec-checkds / dnssec-coverage / dnssec-keymgr に依存しないことを確認（9.18 で削除） | named9, named-checkconf, dig, nslookup, rndc | bos.net.tcp.bind / bind_utils はマイグレーション時に削除される。bind.rte は既定インストール対象外 | S35 |
