# §A. 6 領域の判定詳細

[全体観 §2 「6 領域分類」](../overview/02-domains.md) の各領域について、対象外と判定した理由 (または対象と判定した理由) を述べる。

判定の枠組みは [SSOT ③](../overview/01-ssot.md) (役割境界による分類基準) を適用したもの。

---

## A-1. OS 担当者管轄の 5 領域はすべて対象外

### IBM 保守領域

| 項目 | 内容 |
|---|---|
| 対象範囲 | z/OS の起動時に必須のシステムライブラリ群 (SYS1.NUCLEUS 等) |
| 重要度 | 高 |
| 判定 | 本ツール対象外。SMP/E に管理を委ねる |
| 判定根拠 | 公式マニュアルで「IPL 時に決定され never be changed or removed」と明記 |
| 出典 | S2 (zOS31_ieae200) |
| 根拠強度 | 強 |

### システム制御領域

| 項目 | 内容 |
|---|---|
| 対象範囲 | z/OS の動作を制御する設定パラメータ群 (PARMLIB の IEASYSxx, SMFPRMxx 等) |
| 重要度 | 中 |
| 判定 | 本ツール対象外。ただし運用所管メンバ (COMMNDxx 等) は運用資材領域に含める |
| 判定根拠 | システム部門の運用ルールで管理されているのが一般的 |
| 出典 | S2, S13 (Common Criteria) |
| 根拠強度 | 中 |

### システムプロシージャ領域

| 項目 | 内容 |
|---|---|
| 対象範囲 | z/OS のシステム標準プロシージャ群 (SYS1.PROCLIB) |
| 重要度 | 中 |
| 判定 | 本ツール対象外。運用カスタム PROC は運用資材領域に含める |
| 判定根拠 | JES2 起動時に決定される構造 |
| 出典 | S3 (zOS31_ieab600), S4 |
| 根拠強度 | 強 |

---

## A-2. アプリ部門領域 + 運用資材領域

### アプリ実行領域

| 項目 | 内容 |
|---|---|
| 対象範囲 | アプリのロードモジュール (USER.LOADLIB) |
| 重要度 | 中 |
| 判定 | 本ツール対象外。バイナリ差分はアプリソース領域で追跡することで補える |
| 判定根拠 | RECFM=U で人間可読でなく、Excel で差分を見ても理解不能 |
| 出典 | S6, S11 |
| 根拠強度 | 強 |

### アプリソース領域

| 項目 | 内容 |
|---|---|
| 対象範囲 | アプリ部門のソースコードと JCL |
| 重要度 | 中 |
| 判定 | 本ツール対象外。既存 SCM に管理を委ねる |
| 判定根拠 | SCLM Project Manager Guide で SCM 中核領域として定義 |
| 出典 | S8 |
| 根拠強度 | 強 |

### 運用資材領域 ★本ツールの対象★

| 項目 | 内容 |
|---|---|
| 対象範囲 | 運用部門が直接編集する JCL / REXX / PROC 等。運用所管 PARMLIB メンバ (COMMNDxx 等) を含む |
| 重要度 | 高 |
| 判定 | **本ツールの唯一の対象**。Change Tracker の代替として軽量版を構築 |
| 判定根拠 | S5 / S13 / S17 で公式管理空白を確認。RACF/SMF (S13) は中身差分を追えず |
| 出典 | S5, S13, S17 |
| 根拠強度 | 中 |

---

!!! info "§A の総括"
    OS 担当者管轄 (5 領域) はすべて公式管理体系がある or 触らない。運用部門管轄の運用資材領域だけが公式管理空白で、本ツールが埋める対象。
